# 方案A 原版 vs 增强版 对比

两种方案的**外部协议都不变**，差异仅在内部索引映射逻辑。

---

## 思路区别

| | 原版 | 增强版 |
|------|------|--------|
| 做法 | 一行 sort，固定 l=0,m=1,r=2 | X分组检测共列，动态映射 l/m/r |
| m 语义 | 永远是排序第2个桶 | 共列时=后桶，否则=中列 |
| 复杂度 | 1行代码 | ~30行 |

---

## 情况逐一对比

### 情况1：3列分开（正常）

```
    左(B0)      中(B1)      右(B2)
```

✅ **一致**。所有指令原版/增强版输出相同。

---

### 情况2：共列在左（左2桶，右1桶）

```
    左列              右列
  ┌─────┐
  │ 前桶 │ Y=1.5      ┌─────┐
  │ 后桶 │ Y=2.5      │ 独桶 │
  └─────┘            └─────┘
```

原版排序: [左前, 左后, 右] → l=左前, m=左后, r=右
增强版: 共列在左 → l=左前, m=左后, r=右

✅ **一致**。因为 index 1 恰好就是后桶。

---

### 情况3：共列在右（左1桶，右2桶）🔴 差异

```
    左列              右列
  ┌─────┐           ┌─────┐
  │ 独桶 │           │ 前桶 │ Y=1.5
  └─────┘           │ 后桶 │ Y=2.5
                    └─────┘
```

原版排序: [左, 右前, 右后] → l=左, m=右前, r=右后
增强版映射: 共列在右 → l=左, m=右后, r=右前

| 指令 | 原版 | 增强版 | 差异 |
|------|------|--------|------|
| `ml` | m→l: **右前**→左 | m→l: **右后**→左 | 🔴 第一目标不同 |
| `mr` | m→r: 右前→右后 | m→r: 右后→右前 | 🔴 路径相反 |
| `lm` | l→m: 左→**右前** | l→m: 左→**右后** | 🔴 第二目标不同 |
| `lr` | l→r: 左→右后 | l→r: 左→右前 | 🔴 第二目标不同 |
| `rm` | r→m: 右后→**右前** | r→m: 右前→**右后** | 🔴 第二目标不同 |
| `rl` | r→l: 右后→左 | r→l: 右前→左 | 🔴 第一目标不同 |

**增强版 m 始终指向后桶**，而原版 m 指向右前（排序中间位）。这是核心差异。

---

### 情况4：共列在中间（左1，中2）

```
    左列            中间列
  ┌─────┐         ┌─────┐
  │ 独桶 │         │ 前桶 │ Y=1.5
  └─────┘         │ 后桶 │ Y=2.5
                  └─────┘
```

原版: l=左, m=中前, r=中后
增强版: l=左, m=中后, r=中前

与情况3同类型——原版 m=中前，增强版 m=中后。

---

### 情况5：全1列

原版和增强版都退化为 Y 排序 → l=前, m=中, r=后。

✅ **一致**。

---

## 总结

| 场景 | 原版 vs 增强版 | 关键差异 |
|------|----------------|----------|
| 3列分开 | 一致 | 无 |
| 共列在左 (左2右1) | 一致 | index=1 恰好是后桶 |
| 共列在右 (左1右2) | 🔴 不一致 | m=右前 vs m=右后 |
| 共列在中 (左1中2) | 🔴 不一致 | m=中前 vs m=中后 |
| 全1列 | 一致 | 无 |

**核心差异只在"共列在排序的中间位置"时出现。** 原版 m 是排序 index=1，增强版 m 是后桶——当共列正好排在后面（情况2），二者碰巧一致；当共列排在前面（情况3、4），二者分叉。

---

## 建议

如果实际场景中，两个桶在同列时**总是左列有2个**（或总是右列有2个），原版就够了。如果左右都可能出现共列，增强版的 `m=后桶` 语义更可靠——因为**指令字符的含义与几何事实绑定，不依赖排序的偶然位置**。
