                       各阶段收到 "0" 行为总表
══════════════════════════════════════════════════════════════════════════════════════

  阶段                     │ USB 用过了吗? │ 收到 "0" 的行为
  ────────────────────────┼──────────────┼──────────────────────────────
  USB 预热中 (等指令)       │   ✗ 没有      │  无视，继续等 USB 指令
  USB 检测中 (聚类未完成)    │   ✗ 没有      │  无视，继续检测
  USB 刚完成→D435 切换中    │   — 过渡中    │  不打断切换流程
  D435 预热中 (等 1m/2m)    │   ✓ 用过      │  保持推流，继续等 1m/2m
  D435 检测中 (2m/1m)       │   ✓ 用过      │  暂停检测，D435 继续推流，启动 20s 计时
  D435 暂停中               │   ✓ 用过      │  timer≤20s: 保持暂停，收到 1m/2m 恢复
                             │              │  timer>20s: 释放 D435，进入等待 data.txt 为空
  等待 data.txt 为空 (准备)  │   — 无相机   │  无视（无相机运行，等 data.txt 清空后开 USB）
  新一轮 USB 预热中          │   ✗ 没有     │  无视，进入下一轮循环


══════════════════════════════════════════════════════════════════════════════════════
                       关 键 改 动 点
══════════════════════════════════════════════════════════════════════════════════════

  现状                                   →  改进后
  ───────────────────────────────────────────────────────────────────
  CameraSource 在 Pipeline.run() 内      →  CameraSource 由 Orchestrator
  start，跟随检测逻辑同步启动                提前启动，与检测逻辑解耦

  start_event 同时触发 "推流+检测"        →  推流先于 start_event 启动
                                             start_event 只触发 "开始检测"

  Pipeline 既管相机又管检测               →  Pipeline 只管检测逻辑
                                             相机生命周期由 Orchestrator 管

  c=0 时关相机并写 "0"                   →  用过 USB 前: c=0 是无视的
                                             用过 USB 后: c=0 暂停检测，相机不关

  新增状态标记 `_usb_used`               →  用来区分 "0 该无视还是暂停"

  新增 D435 暂停超时 (20s)                →  超时=任务结束，释放 D435，_usb_used
                                             重置为 False，回到初始等 USB 指令

                       时 序 对 比
══════════════════════════════════════════════════════════════════════════════════════

  改前:
  ─────
  指令到达 → 开相机(白平衡3s) → 开始检测  ← 前3s 画面过曝/偏白，检测失败
  0 模式 → 关相机                        ← 重新开又得等白平衡

  改后:
  ─────
  程序启动 → 开相机(后台预热) → ...等指令... → 开始检测  ← 首帧即稳
  0 模式 → 暂停检测，相机继续推流            ← 恢复时不需要重新预热
