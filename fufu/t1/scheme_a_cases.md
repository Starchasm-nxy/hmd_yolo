# 方案A 枚举：XY 二级排序下 6 指令映射

## 排序规则

```python
# X 为主键（量化到 THRESHOLD 网格），Y 为副键（小=近=前）
camera_centers.sort(key=lambda c: (round(c[0] / THRESHOLD), c[1]))
```

同一网格 = 同列，同列内按 Y（深度）从小到大 = 前→后。

---

## 情况 1：3 个桶各占一列（正常情况）

```
     B0          B1          B2
  X=-1.2       X=0.1       X=1.4
  Y=2.0        Y=1.8       Y=2.2

round(X/THRESHOLD):  -3格    0格    3格   → 全部不同

排序: [B0, B1, B2]
索引: 0=左      1=中      2=右
```

| 指令 | 索引 | 含义 |
|------|------|------|
| `ml` | (1,0) | 中→左 |
| `mr` | (1,2) | 中→右 |
| `lm` | (0,1) | 左→中 |
| `lr` | (0,2) | 左→右 |
| `rm` | (2,1) | 右→中 |
| `rl` | (2,0) | 右→左 |

✅ 完全不变。

---

## 情况 2：2 列 — 左列 2 个，右列 1 个

```
    左列              右列
  ┌─────┐
  │ B0  │ Y=1.5 (前)
  │     │
  │ B1  │ Y=2.5 (后)
  └─────┘           ┌─────┐
  X≈-0.8            │ B2  │ X=1.3 Y=2.0
                    └─────┘

round(X/THRESHOLD): B0和B1同格, B2不同格

排序: [B0(左前), B1(左后), B2(右)]
索引: 0=左前    1=左后    2=右
```

| 指令 | 索引 | 含义 |
|------|------|------|
| `ml` | (1,0) | **左后→左前** ← 同列后→前 |
| `mr` | (1,2) | 左后→右 |
| `lm` | (0,1) | **左前→左后** ← 同列前→后 |
| `lr` | (0,2) | 左前→右 |
| `rm` | (2,1) | 右→左后 |
| `rl` | (2,0) | 右→左前 |

⚠️ `ml` 和 `lm` 变成了**同列内的前后遍历**，不再是跨列。

## 情况 3：2 列 — 左列 1 个，右列 2 个

```
    左列              右列
  ┌─────┐           ┌─────┐
  │ B0  │ X=-1.0   │ B1  │ Y=1.5 (前)
  └─────┘           │     │
                    │ B2  │ Y=2.5 (后)
                    └─────┘
                    X≈1.2

排序: [B0(左), B1(右前), B2(右后)]
索引: 0=左     1=右前     2=右后
```

| 指令 | 索引 | 含义 |
|------|------|------|
| `ml` | (1,0) | 右前→左 |
| `mr` | (1,2) | **右前→右后** ← 同列前→后 |
| `lm` | (0,1) | 左→右前 |
| `lr` | (0,2) | 左→右后 |
| `rm` | (2,1) | **右后→右前** ← 同列后→前 |
| `rl` | (2,0) | 右后→左 |

⚠️ `mr` 和 `rm` 变成了**同列内的前后遍历**。

## 情况 4：3 桶挤在 1 列（全同列）

```
    同一列
  ┌─────┐
  │ B0  │ Y=1.5 (前)
  │ B1  │ Y=2.0 (中)
  │ B2  │ Y=2.5 (后)
  └─────┘
  X≈-0.6~-0.9

排序: [B0(前), B1(中), B2(后)]
索引: 0=前      1=中      2=后
```

| 指令 | 索引 | 含义 |
|------|------|------|
| `ml` | (1,0) | 中→前 |
| `mr` | (1,2) | 中→后 |
| `lm` | (0,1) | 前→中 |
| `lr` | (0,2) | 前→后 |
| `rm` | (2,1) | 后→中 |
| `rl` | (2,0) | 后→前 |

⚠️ 全部变为同列内的深度方向遍历，l/m/r 语义退化为前/中/后。

## 情况 5：2 列但中间桶 X 居中（左列1，中列2）

```
    左列            中间列
  ┌─────┐         ┌─────┐
  │ B0  │ X=-1.2 │ B1  │ Y=1.5 (前)
  └─────┘         │     │
                  │ B2  │ Y=2.5 (后)
                  └─────┘
                  X≈0.0

排序: [B0(左), B1(中前), B2(中后)]
索引: 0=左     1=中前     2=中后
```

| 指令 | 索引 | 含义 |
|------|------|------|
| `ml` | (1,0) | 中前→左 |
| `mr` | (1,2) | **中前→中后** |
| `lm` | (0,1) | 左→中前 |
| `lr` | (0,2) | 左→中后 |
| `rm` | (2,1) | **中后→中前** |
| `rl` | (2,0) | 中后→左 |

## 情况 6：边界抖动 — THRESHOLD 选值影响

```
  B0 和 B1 实际 X 差只有 0.1m，但跨了网格边界

  THRESHOLD=0.3:
    B0 X=-0.55 → round(-1.83)=-2 → grid=-0.6
    B1 X=-0.45 → round(-1.50)=-2 → grid=-0.6  ← 同格 ✓
    B2 X= 0.80 → round( 2.67)= 3 → grid= 0.9

  THRESHOLD=0.5:
    B0 X=-0.55 → round(-1.10)=-1 → grid=-0.5
    B1 X=-0.45 → round(-0.90)=-1 → grid=-0.5  ← 同格 ✓
    B2 X= 0.80 → round( 1.60)= 2 → grid= 1.0

  THRESHOLD=0.8:
    B0 X=-0.55 → round(-0.69)=-1 → grid=-0.8
    B1 X=-0.45 → round(-0.56)=-1 → grid=-0.8  ← 同格 ✓
    B2 X= 0.80 → round( 1.00)= 1 → grid= 0.8
```

✅ 选 THRESHOLD ≥ 0.5 即可避免大部分边界问题。桶间距通常 > 0.5m（不同列）或 < 0.2m（同列）。

## 总结

| 桶排列 | 0 语义 | 1 语义 | 2 语义 | 失效指令 |
|--------|--------|--------|--------|----------|
| 3列分开 | 左 | 中 | 右 | 无 |
| 2列(左2右1) | 左前 | 左后 | 右 | ml/lm 退化为同列前后 |
| 2列(左1右2) | 左 | 右前 | 右后 | mr/rm 退化为同列前后 |
| 全1列 | 前 | 中 | 后 | 全部退化为深度方向 |
| 2列(左1中2) | 左 | 中前 | 中后 | mr/rm 退化为同列前后 |

**结论**：方案 A 不需要改指令协议。当出现同列多桶时，`m`/`l`/`r` 代表的索引仍然有效——只是语义从"空间列"变成了排序位置。下位机只要按 `gaozhi.txt` 输出的坐标顺序去飞，就不需要关心背后是"左中右"还是"前后中"。
